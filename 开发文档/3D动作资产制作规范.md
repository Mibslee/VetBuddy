# 3D 动作资产制作规范

## 文件格式

- **格式**: USDZ (Apple 通用场景描述压缩包)
- **来源工具**: Blender → USDZ 导出，或 Reality Composer 直接制作
- **备选**: glTF 2.0 → Reality Converter 转 USDZ

## 资产规格

| 属性 | 要求 |
|------|------|
| 多边形数 | ≤ 50,000 tris（移动端性能） |
| 纹理尺寸 | 1024x1024 最大，推荐 512x512 |
| 动画帧率 | 30 fps |
| 动画时长 | 一个完整动作循环（3-8秒） |
| 文件大小 | ≤ 5MB / 个 |
| 骨骼 | 建议使用简单骨骼（≤ 30 bones） |
| 场景原点 | 脚底中心，Y轴朝上 |

## 人物模型要求

- 老年角色形象，体型适中
- 穿着简单运动服（避免复杂布料模拟）
- 面部简化处理（低多边形或无面部细节）
- 肤色和服装颜色参考设计系统：
  - 服装主色: `#C73E1D`（赤陶红）或 `#F5F0E1`（宣纸色）
  - 肤色: 中性暖色调

## 集成方式

### 1. 文件放置

```
Resources/
  Models/
    sit_to_stand.usdz
    calf_raise.usdz
    ...
```

### 2. SwiftUI 加载 (RealityKit)

```swift
import RealityKit
import SwiftUI

struct ExerciseModelView: View {
    let modelName: String

    var body: some View {
        Model3D(named: modelName, bundle: .main) { model in
            model
                .resizable()
                .scaledToFit()
        } placeholder: {
            ProgressView()
        }
    }
}
```

### 3. 替换 ExerciseDemoView 占位符

当前 `ExerciseDemoView` 中的 `exercisePlaceholder` 是纯色占位。
替换为 USDZ 加载后，效果如 DESIGN.md 中的 _3/code.html 所示：
- 白色圆角卡片背景
- 3D 模型居中显示
- 底部叠加渐变遮罩
- 中央播放/暂停按钮

### 4. RealityKit 动画控制

```swift
// 自动循环播放
model.playAnimation(model.availableAnimations.first?.repeat())

// 暂停/恢复
model.pauseAnimation()
model.resumeAnimation()
```

## 制作流程建议

1. **建模**: Blender 中创建老年角色 + 运动服装
2. **骨骼绑定**: 使用 Mixamo 自动绑定（免费）
3. **动画**: 手动 K 帧或使用 Mixamo 动作库（需适配）
4. **导出**: Blender → File → Export → USD (.usdz)
5. **验证**: 用 macOS Quick Look 或 Reality Composer 打开检查
6. **优化**: 减面、压缩纹理、确保 ≤ 5MB

## 替代方案（如果 3D 资产制作周期长）

### 方案 A: Lottie 动画
- 用 After Effects 制作 2D 简笔动画
- 导出为 JSON (Lottie 格式)
- 用 lottie-ios 库播放
- 优点：制作快，文件小；缺点：无 3D 效果

### 方案 B: 视频演示
- 真人演示录制 → 压缩为 MP4
- 用 AVPlayer 播放
- 优点：最真实；缺点：文件大，需要拍摄

### 方案 C: 精灵序列帧
- 逐帧绘制或 3D 渲染为 PNG 序列
- 用 SpriteKit 播放
- 优点：兼容性好；缺点：文件较大
